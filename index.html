索引入门

数据库的优化第一步应该想到的就是索引

MySQL常用的索引有两种

  • B+树
  • Hash（InnoDB不支持）

创建索引&删除索引&查看索引

索引优化前置知识：磁盘预读

也就是不把数据全部读入内存，只读取部分页。

索引是什么

  • 索引是帮助MySQL高效获取数据的数据结构
  • 索引存储在文件系统中
  • 索引的文件存储形式与存储引擎有关
  • 索引文件的结构
    • hash
    • 二叉树
    • B树
    • B+树

为什么最终会选择B+树？

不使用哈希的原因

hash表的索引是使用hash算法+数组+联表+红黑树的方式实现的。

缺点：

  1. 利用hash存储的话需要将所有的数据文件添加到内存，比较浪费内存空间
  2. 如果所有的查询都是等值查询，hash确实很快，但是实际中范围查找比等值查找多得多，因此hash不适合
  3. 会产生hash冲突（使用扰动函数减少hash冲突）

而内存中自己使用的是hash因为它本身在内存中，而且也速度快，而数据库是在磁盘中，需要进行慢速的交互的，所以这个时候用hash就不合适了。

不使用二叉树的原因

无论是二叉树还是红黑树，都会因为树的深度过深而造成IO次数变多，影响读写效率，这是为什么要少用select * 的原因，select * 会扫描全盘，降低IO速度。

为什么B树可以

  1. B树的每个节点都有多个数据，减少IO
  2. B树的深度更潜，减少IO

为什么最终使用B+树

因为B树的非叶子结点中存储数据会导致B树能够容纳的范围变小。

优势：

  1. B+树在相同内存空间中能够容纳更多的数据
  2. B+树有两种头指针，一个指向根节点一个指向最小叶子结点，所以既可以范围查找，又可以从根节点开始随机查找

B+树到底是3层还是4层？

取决于索引的类型，一个节点能够容纳的范围等于，节点大小除以索引的大小，所以索引越小，能够容纳的越多，层数也就越小，所以多建议使用int来做索引而不是varchar或text。

前缀索引

​ 有时候需要索引很长的字符串，这会让索引变大且慢，通常情况下可以使用某个列开始的部分字符串，这样大大的节约索引空间，从而提高索引效率，但这会降低索引的选择性，索引的选择性是指不重复的索引值和数据表记录总数的比值，索引的选择性越高则查询效率越高，因为选择性高的索引可以让，mysql在查找的时候过滤掉更多的行。

​ 一般情况下某个列前缀的选择性也是足够高的，足以满足查询的性能，但是对应BLOB，TEXT，VARCHAR类型的列，必须使用前缀索引，因为mysql不允许索引这些列的完整长度，使用该方法的诀窍在于要选择足够长的前缀以保证较高的选择性，通过又不能太长。

添加前缀索引

==注意：前缀索引是一种能使索引最小最块的有效方法，但是也包含缺点：mysql无法使用前缀索引做order by和group by。==

不同数据库引擎在B+树叶子节点中存放的内容不同

  • InnoDB叶子节点直接放置整行数据
    • InnoDB默认对主键创建索引，如果没有主键，那么会选择唯一键，如果还没有，就会生成一个6字节的row_id作为主键，此时叶子节点是整行数据
    • 如果创建索引的键是其他字段，那么叶子结点存放的是该记录的主键，然后通过主键索引找到对应的记录。这个叫==回表==。数据量小的时候回表负优化，数据量大才有用。
  • MyISAM存放的是一个指针，指针指向的位置为表的数据

不同主键策略对索引的影响

​ 如果是单体系统，那么推荐使用主键自增，这样会减少索引的维护开销，如果使用随机生成的话，如果一直往中间区域插入的话，会一直产生页分裂，索引的维护开销会很大。

​ 如果是分布式系统，一般使用雪花算法生成主键

一些额外的专有名词

  • 回表：查多次索引

  • 索引覆盖：不触发回表，索引查主键，或主键查整列

  • 最左匹配：在使用组合索引的时候，只有匹配到第一个字段才会走索引如下

  • 索引下推：mysql中数据的拉取过程如下

    ==用户—>mysql server—–>存储引擎—->磁盘==

    考虑这样一条sql：select * from table where name=? and age=?

    如果没有索引下推就会按照上述的流程，如果有，那么拉取数据的时候直接根据name，age来获取数据，不需要server做任何的数据筛选。这样会减少IO量。

    ==缺点==：需要再磁盘上多做数据筛选，原来的筛选是放在内存中的，现在放在磁盘，看起来成本很高，但是数据是排序的，数据是聚集的，所以性能不会有影响，而且整体IO量会少，性能会提升。

  • 谓词下推：

    考虑这样一条sql：select t1.name,t2.name from t1 join t2 on t1.id=t2.id;

    这条sql有两种做法：

    • 现做表连接，组合出一个中间表，然后再查询需要的列
    • 先把需要的列都拿出来，然后再做关联：谓词下推，IO量少，效率高

解释一下什么是MMR？

MMR：mult_range read

在数据回表的时候，我们根据第一个B+树获取到了id，然后通过排序来执行范围查询，避免了一个一个查找。

解释一下什么是FIC？

FIC：fast index create

在删除或插入数据的时候，需要更新索引，更新索引的过程是会先创建一个新的表，然后把所有数据插到新表，并创建索引，然后删除原表。有了FIC之后，会先给表上一个锁，读不影响，写会被影响。

索引分类

  • 主键索引
  • 唯一索引
  • 普通索引：给除主键、唯一键之外的其他字段创建的索引，也叫辅助索引或二级索引
  • 全文索引：一般不用而是用ES
  • 组合索引

索引匹配方式

  1. 全值匹配：和索引中的所有列进行匹配，出现在组合索引中。
  2. 匹配最左前缀：只匹配索引中的前面几个字段
  3. 匹配列前缀：匹配某一列的值的开头部分explain select * from staffs where name like 'J%'
  4. 匹配范围值：可以查找某一范围的数据explain select * from staffs where name >'Mary'
  5. 精确匹配某一列并范围匹配另外一列：可以查询第一列的全部，和第二列的部分explain select * from staffs where name='Mary' and age > 25，==范围查找后的索引会失效==
  6. 只访问索引的查询：查询的时候只需要访问索引，不需要访问数据行，本质上就是覆盖索引。explain select name,age,pos from staffs where name='Mary' and age=25 and pos='Beijin'

哈希索引

  1. 基于哈希表的实现，只有精确匹配索引所有类的查询才有效
  2. 在mysql中，只有memory的存储引擎显式支持哈希索引
  3. 哈希索引自身只需存储对应的hash值，所以索引的结构十分紧凑，这让哈希索引查找的速度非常快

==限制==：

  1. 哈希索引只包含哈希值和行指针，而不存储字段值，索引不能使用索引中的值来避免读取行
  2. 哈希索引数据并不是按照索引值顺序存储的，所以无法进行排序
  3. 不支持部分列匹配，因为hash值使用全部内容来计算的
  4. 不支持任何范围查找
  5. 访问速度非常快，但是当有很多冲突的时候，要遍历所有的行指针
  6. 哈希冲突多的话，维护代价也会很高

==案例：==

当需要存储大量的URL时，可以用CRC32做哈希，CRC32会把URL编码为10位的数字，这样最后的索引体量会小很多，查询效率会很高。

组合索引

当包含多个列作为索引，需要注意的事正确的顺序依赖于该索引的查询，同时需要考虑如何更好的满足排序和分组的需要。

==or在组合索引中的特殊性==：如果组合索引包含全部字段，并使用了or，那么索引生效，并且使用的是整个组合索引；如果组合索引不包含全部字段，并使用了or，那么索引不生效，但是如果只查询索引列，又会生效。

聚簇索引与非聚簇索引

  • 聚簇索引（InnoDB）：不是单独的索引类型，而是一种输出存储方式，指的事数据行跟相邻的键值紧凑的存储在一起
  • 非聚簇索引（MyISAM）：数据文件跟索引文件分开存放

Select语句

搜索列，对列结果进行操作，起别名：

select ename, sal * 12 as ‘annule_sal’ from emp;

处理空值，空值不是0，空值参与运算返回空值：

select

​ enam ,

​ sal * 12 as ‘annule_sal’ + (case when comm is NULL then 0 else comm end) as ‘annule_sal’

from

​ emp;

去重，如果后面跟多个字段，则是对组合去重

select distinct deptno from emp;

where子句

不等于

select * from emp where deptno <> 10;

范围

select * from emp where sal between 800 and 1500;

select * from emp where sal>=800 and sal<=1500;

判空

select * from emp where comm is null;

select * from emp where comm is not null;

或

select * from emp where sal in (800,1500);

select * from emp where sal==800 or sal==1500;

模糊查询，名字中包含A的，名字中有A且前面只有一个字母的

select * from emp where ename like ‘%A%’;

select * from emp where ename like ‘_A%’;

order by子句

按照某一列排序，倒序

select * from emp order by sal;

select * from emp order by sal desc;

与where混用

select * from emp where deptno<>10 order by sal desc;

多列排序，先按deptno升序排列，deptno相同的按ename降序

select * from emp order by deptno ASC, ename DESC;

常用函数

  • 转大小写

  • 字符串拼接

  • 字符串长度

  • 截取子串

select substring(‘goodgoodstudy’,0,5)

  • 去掉左边的空串，右边的空串，两边的空串

select lttim/rtrim/trim(‘ aa bb ‘);

  • 绝对值，向上取整，向下取整

select abs(), ceil(), floor();

  • 在第几位小数四舍五入

select round(3.14159,3); //3.142

  • 保留3位小数

select truncate(3.14159,3)

  • 时间

select curdate(); //日期

select curtime(); //时间

select now(); //日期加时间

  • 取月份，取年份

select MONTH(hiredate) from emp;

select ename, hiredate from emp order by year(hiredate)

组函数

一组数据，最后输出一条值，叫组函数，max min avg sum count，null在count中不计数

select max(sal) from emp;

组函数不可被当做组合查询列，因为max(sal)只返回1行，而最大工资的可能有多行：

select ename, max(sal) from emp; x

组函数只能当做子查询

select ename from emp where sal = (select max(sal) from emp);

group by子句

按照部门分组，然后从每组中选出最大薪水

select deptno, max(sal) from emp group by deptno;

利用上面的生成的中间表，查出每个组中薪水最大的

select ename from emp join

(select deptno, max(sal) from emp group by deptno) t

where emp.deptno = t.deptno and emp.sal = t.sal;

having子句

对group by之后的内容进行限制，选出平均薪水大于2000的部分

select avg(sal), deptno from emp group by deptno having avg(sal) > 2000;

select总结

取数据，过滤，分组，分组限制，排序

多线程

程序、进程、线程

  • 程序：是为完成特定任务、用某种语言编写的一组指令的集合，是一段静态的代码。
  • 进程：是程序的一次执行过程。正在运行的一个程序，进程作为资源分配的单位，在内存中会为每个进程分配不同的内存区域。进程是动态的，有生命周期，有自身的产生、存在、消亡的过程。
  • 线程：进程可进一步细化为线程，是一个程序内部的一条执行路径。若一个进程同一时间并行执行多个线程，就是支持多线程的。

并行和并发

  • 并行：多个CPU同时执行多个任务
  • 并发：一个CPU“同时”执行多个任务

synchronized中的参数问题

synchronized中传入的参数就是我们要锁的对象，如果我们的线程对象实现的是Runnable接口，那么每个线程调用的都是同一个实现类，所以我们在synchronized中传入this就可以将资源锁住。但是如果我们的线程对象是继承的Thread对象，那么我们new的每一个线程都是不一样的，这时如果我们任然再synchronized中传入this就会失效。

一般情况下我们会传入这个类的字节码信息，因为字节码是唯一的，传入的东西叫做同步监视器。

总结：

  1. 必须是引用数据类型，不能是基本数据类型
  2. 也可以创建一个专门的同步监视器，没有任何业务意义
  3. 一般使用共享资源做同步监视器即可
  4. 在同步代码块中不能改变同步监视器的引用
  5. 尽量不要String和包装类Integer做同步监视器
  6. 建议使用final修饰同步监视器
  7. 同步代码块中是会发生CPU切换的，只不过锁还在，其他线程还是不能访问同步代码块
  8. 多个代码块使用了同一个同步监视器，锁住一个代码块的同时，也锁住所有使用该锁的代码块，其他线程无法访问其中任何一个代码块

同步方法

将有线程安全的代码块抽出作为独立的方法，然后加上synchronized修饰。

而对于继承Thread类的，就需要再方法上在加上static修饰符，保证唯一性。

总结：

  1. 不要将run()定义为同步方法，因为锁住太多东西了，降低效率
  2. 非静态同步方法的同步监视器就是this，静态同步方法的事类名.class字节码信息对象
  3. 同步代码块的效率要高于同步方法，因为同步方法是将线程挡在了方法的外部，而同步代码块锁将线程挡在了代码块的外部，但是却是方法的内部
  4. 同步方法的锁是this，一旦锁住了一个方法就锁住了所有的同步方法；同步代码块只是锁住使用该同步监视器的代码块，而没有锁住其他监视器的代码块

Lock锁

与synchronized相比，lock可提供多种锁方案，更灵活。synchronized是Java关键字，这个关键字的识别是靠Java来识别完成的，是虚拟机级别的。

但是Lock锁是API级别的，提供了相应的接口和对应的实现类，这个方式更灵活，表现出来的性能优于之前的方式。

总结：

  • Lock是显示锁，synchronized是隐示锁

  • Lock只有代码块锁

  • 使用Lock锁Jvm将花费更少的时间来调度线程，性能更好，并且有更好的扩展性（有很多子类）

重点

在Java对象中，有两种池

  • 锁池
  • 等待池

如果一个线程调用了某个对象的wait方法，那么该线程进入到该对象的等待池中(并且已经将锁释放)，如果未来的某一个时刻，另一个线程调用相同对象的notify方法，或者notifyAll方法，那么该等待池中的线程就会被唤醒，然后进入到对象的锁池里面去获得该对象的锁，如果获得锁的成功后，那么该线程就会沿着wait方法之后的路径继续执行。注意是沿着wait方法之后。

注意：

  • wait方法和notify方法只有放在同步方法或同步块中才生效
  • wait释放锁，sleep不释放锁

Condition

它用来替代传统的Object的wait、notify实现线程间的协作，相比使用Object的wait、notify使用Condition的await、signal这种方法实现线程间的协作更加的安全和高效。

它的更强大的地方在于：能够更加精细的控制多线程的休眠与唤醒。对于同一个锁，我们可以创建多个Condition，在不同的情况下使用不同的Condition

一个Condition包含一个等待队列。一个Lock可以产生多个Condition，所以可以有多个等待队列。

在Object的监视器模型上，一个对象拥有一个同步队列(持有锁)和一个等待队列(不持有锁)，==而Lock拥有一个同步队列和多个等待队列。==

Object中的wait、notify、notifyAll方法是和同步锁synchronized捆绑使用的；而Condition是需要与互斥锁、共享锁捆绑使用。

调用Condition的await、signal、signAll方法，都必须在lock保护子类，就是说必须在lock.lock()和lock.unlock()之间可以使用

  • Condition中的await对应Object的wait
  • Condition中的signal对应Object的notify

数据链路层

基本概念

数据链路层负责通过一条链路从一个结点向另一个物理链路直接相连的相邻结点传送数据报。

  • 结点：主机、路由器
  • 链路：网络中两个结点之间的物理通道，链路的传输介质主要有双绞线、光纤和微波。分为有线链路、无线链路。
  • 数据链路：网络中两个结点之间的逻辑通道，把实现控制数据传输协议的硬件和软件加到链路上就构成数据链路。
  • 帧：链路层的协议数据单元，封装网络层数据报。

功能概述

数据链路层在物理层提供服务的基础上向网络层提供服务，其最基本的服务是将源自网络层来的数据可靠地传输到相邻结点的目标机网络层。其主要作用是加强物理层传输原始比特流的功能，将物理层提供的可能出错的物理连接改造成为逻辑上无差错的数据链路，使之对网络层表现为一条无差错的链路。

  • 功能一：为网络层提供服务。无确认无连接服务，有确认无连接服务，有确认面向连接服务。有连接一定有确认！
  • 功能二：链路管理，即连接的建立、维持。释放(用于面向连接的服务)。
  • 功能三：组帧。
  • 功能四：流量控制。限制发送方
  • 功能五：差错控制(帧错/位错)

中继器

  • 诞生原因：由于存在损耗，在线路上传输的信号功率会逐渐衰减，衰减到一定程度时将造成信号失真，因此会导致接受错误。

  • 中继器的功能：对信号进行再生和还原，对衰减的信号进行放大，保持与原数据相同，以增加信号传输的距离，延长网络的长度。

  • 中继器的两端：两端的网络部分是网段，而不是子网，适用于完全相同的两类网络的互连，且两个网段速率要相同。中继器只将任何电缆段上的数据发送到另一段电缆上，他仅作用于信号的电气部分，并不管数据中是否有错误数据或不适于网段的数据。

    两端可连相同媒体，也可连不同媒体。

    中继器两端的网段一定要是同一协议。(中继器不会存储转发)

  • 5-4-3规则：网络标准中都对信号的延迟范围作了具体的规定，因而中继器只能在规定的范围内进行，否则会故障

集线器(多口中继器)

集线器的功能：对信号进行再生放大转发，对衰减的信号进行放大，接着转发到其他所有(除输入端口处)处于工作状态的端口上，以增加信号传输的距离，延长网络的长度，不具备信号的定向传送能力，只能进行广播，是一个共享设备。

奈氏准则与香农定理

奈氏准则与香农定理

失真

信道带宽：是信道能通过的最高频率和最低频率之差。

柰氏准则

在理想低通(无噪声，带宽受限)条件下，为了避免码间串扰(频率过高而无法分辨码元)，极限码元传输速率为2 W Baud，W是信道带宽，单位是Hz（一般为bit/s）。

极限数据率：理想低通信道下的极限数据传输率 = 2 W log V(b/s)

W : 带宽（Hz）

V：几种码元/码元的离散电平数目

  • 1.在任何信道中，码元传输的速率是有上限的，若传输速率超过此上限，就会出现严重的码间串扰问题，使接收端对码元的完全正确识别成为不可能。
  • 2.信道的频带越宽(即能通过的信号高频分量越多)，就可以用更高的速率进行码元的有效传输
  • 3.奈氏准则给出了码元传输速率的限制，但没有对信息传输速率给出限制。
  • 4.由于码元的传输速率受柰氏准则的制约，所以要提高数据的传输速率，就必须设法是每个码元能携带更多的比特的信息量，这就需要采用多元制的调制方法。

香农定理

噪声存在于所有的电子设备和通信信道中。由于噪声随机产生，他的瞬时值有时会很大，因此噪声会使接收端对码元的判决产生错误。但是噪声的影响是相对的，若信号较强，那么噪声影响相对较小。因此，信噪比就很重要。

信噪比 = 信号的平均功率/噪声的平均功率，常记为S/N，并用分贝（dB）作为度量单位，即信噪比（dB） = 10 log10 （S/N）

  • 以dB为单位的信噪比，和S/N的信噪比只是单位不同而已。

香农定理：在带宽受限且有噪声的信道中，为了不产生误差，信息的数据传输速率有上限。

信道的极限数据传输速率 = W log2(1+S/N) (b/s)

  • 1.信道的带宽或信道中的信噪比越大，则信息的极限传输速率就越高。
  • 2.对一定的传输带宽和一定的信噪比，信息传输速率上限就确定了。
  • 3.只要信息的传输速率小于信道的极限传输速率，就一定能找到某种方法实现无差错的传输。
  • 4.香农定理得出的为极限信息传输速率，实际信道能到达的传输速率要比它低不少。

“Nice”和”香浓”

奈氏准则内忧

  • 受自身带宽限制
  • 想要提高数据率，就要提高带宽/采用更多进制的码元

香农定理外患

  • 受带宽和噪声的限制
  • 想要提高数据率，就要提高带宽和信噪比

码元、波特、速率、带宽

码元

码元是指用一个固定时长的信号波形(数字脉冲)，代表不同离散数值的基本波形，是数字通信中数字信号的计量单位，这个时长内的信号称为K进制码元，而该时长称为码元宽度。当码元的离散状态有M个时，此码元为M进制码元。

速率、波特、带宽

速率也叫数据率，是指数据的传输速率，表示单位时间内传输的数据量。可以用码元传输速率和信息传输速率表示。

  • 1、码元传输速率：别名码元速率，表示单位时间内数字通信系统所传输的码元个数（也可以称为脉冲个数或信号变化的次数），单位是波特（baud）。1波特表示数字通信系统每秒传输一个码元。这里的码元可以是多进制码元，但码元速率与进制无关。一秒传输多少个码元
  • 2、信息传输速率：别名比特率，表示单位时间内数字通信系统传输的二进制码元个数（即比特数）。一秒传输多少个比特

关系：若一个码元携带n bit的信息量，则M Baud的码元传输速率所对应的信息传输速率为M * n bit/s

比如，现在有个四进制码元用 00 01 10 11 表示四种信号量，所以一个码元携带2 bit 的信息量，5 Baud的码元传输速率对应的比特率为 2 * 5 = 10b/s

  • 带宽：表示单位时间内从网络的某一点到另一点所能通过的“最高数据率”，常用来表示网络的通信线路所能传输数据的能力。单位是b/s

练习题

由四进制码元可知，一个码元携带2 b的数据

码元的传输速率：8000/4 = 2000 Baud

信息传输速率：2000*2 = 4000 b/s

由十六进制码元可知，一个码元携带4 b的数据

码元传输速率：7200/6 = 1200 Baud

信息传输速率：1200 * 4 = 4800 b/s
